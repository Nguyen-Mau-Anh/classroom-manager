## Design Philosophy

### Core Principles

| Principle | Description | Application |
|-----------|-------------|-------------|
| **Glanceable** | Users should get key info in < 5 seconds | "Next Class" prominent on all dashboards |
| **Role-Focused** | Each dashboard serves one role exceptionally | Admin â‰  Teacher â‰  Student views |
| **Calm Technology** | Reduce cognitive load, surface only what matters | Alerts only for actionable items |
| **Mobile-Ready** | Works on tablets in classrooms | Responsive, touch-friendly |
| **Accessible** | WCAG 2.1 AA compliance | Contrast, focus states, screen readers |

### Design Goals by User

| User | Primary Goal | Design Focus |
|------|--------------|--------------|
| **Admin** | Control & Overview | Dashboard with alerts, quick actions |
| **Teacher** | Know what's next | Large "Next Class" card, simple navigation |
| **Student** | Find my class | Schedule-centric, mobile-first |

## Responsive Behavior

### Breakpoint Adaptations

| Breakpoint | Sidebar | Navigation | Cards |
|------------|---------|------------|-------|
| Desktop (1024px+) | Full sidebar | Top + Side | Grid 3-4 columns |
| Tablet (768px) | Collapsed icons | Top + Side | Grid 2 columns |
| Mobile (< 768px) | Hidden | Bottom tabs | Stack 1 column |

### Mobile-Specific Patterns

- **Bottom navigation** for students (most mobile users)
- **Swipe gestures** for schedule navigation
- **Pull-to-refresh** for schedule updates
- **Touch-friendly** targets (min 44px)

---

## Accessibility Checklist

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | 4.5:1 minimum for text, 3:1 for large text |
| Focus indicators | 2px blue outline on all interactive elements |
| Keyboard navigation | Full tab navigation, arrow keys in lists |
| Screen reader | ARIA labels, landmarks, live regions for updates |
| Reduced motion | Respect `prefers-reduced-motion` |
| Text scaling | Supports 200% zoom without horizontal scroll |
